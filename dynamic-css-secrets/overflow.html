CSS variable inheritance

  • var() in low specificity rules → widely available effects
  • Inheritance can get in the way
  • Prevent inheritance with `--foo: initial`
- Regular properties: Are scoped on the element itself, can be set via CSS rules or inline styles or even with JS - We can provide fallback values on a case by case basis but that's usually repetitive - Pseudo-private variables are a convention: prefix custom properties by `_` to indicate they are for internal use. Useful for baking in fallback values, but also other things. - Provide encapsulation by exposing a higher level styling API. We can change the underlying CSS and the API we've exposed will continue to work
